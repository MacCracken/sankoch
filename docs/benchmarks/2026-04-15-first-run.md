## Notes

- Ratios match C zlib almost exactly (within 1-2 bytes on 4KB text)
- ~10-13x slower than C zlib — expected for a non-optimizing compiler
  with no SIMD, no loop unrolling, byte-at-a-time I/O
- LZ4 throughput: ~170 MB/s compress, ~330 MB/s decompress
- DEFLATE decompress is 5-6x faster than compress
- Random data: DEFLATE compress 3x slower (more hash chain walking, no matches)
- All benchmarks use heap-allocated buffers (stack arrays cause corruption
  at these sizes in Cyrius)
- Two DEFLATE decompress benchmarks skipped (raw DEFLATE without wrapper
  has state interference between compress→decompress; works through
  zlib/gzip wrappers)
